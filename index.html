(Lenguaje de marcas)

Definiciones

Lenguaje de marcas:

Es una forma de definir la estructura del texto o su presentación a través de etiquetas o marcas. Este tipo de lenguajes son los que entienden los navegadores. HTML y XML, son los mas conocidos. A diferencia de los lenguajes de programación, el lenguaje de marca no tiene funciones aritméticas o variables.
(https://fp.uoc.fje.edu/blog/que-es-el-lenguaje-de-marca-o-marcado-html-y-xml/)

Sistemas de gestión de información:

Un sistema de gestión de información (MIS, por sus siglas en ingles) es un conjunto de sistemas y procedimientos que recopilan informacion de una variedad de fuentes, la compilan y la presentan en un formato legible. (https://www.ecured.cu/Sistema_de_Gesti%C3%B3n_de_Informaci%C3%B3n)

Sistema de gestión empresarial:

El termino ERP se refiere a Enterprise Resource Planning, que significa sistéma de planificación de recursos empresariales. Estos programas se hacen cargo de distintas operaciones internas de una empresa, desde producción a distribución o incluso recursos humanos. El ERP funciona como un sistema integrado, y aunque pueda tener menus modulares, es un todo. Es decir, es un único programa con acceso a una base de datos centralizada. Los datos se dan de alta solo una vez y son consistentes, completos y comunes.
(https://www.aner.com/que-es-un-erp.html)

Características

HTML

Una de las ventajas y características del código fuente html o Lenguaje de Marcas es que lo podermos editar o crear desde cualquier editor de texto básico o blog de notas. Siempre que sepas como modificar el formato HTML de una pagina web.
1. Puede ser creado y editado con cualquier editor básico de textos.
2. Es múltiplataforma, debido que todos los navegadores son compatibles.
3. No diferencia entre mayúsculas y minúsculas.
4. Utiliza etiquetas o marcas, que consisten en breves instrucciones de comienzo y final, mediante las cuales se determina la forma en la que debe aparecer en su navegador el texto asi como tambien las imagenes y demas elementos en la pantalla del ordenador.
5. Es un lenguaje estático.
6. Es utilizado para la creación de paginas web.
7. Es un estándar reconocido por todo el mundo y cuyas normas define un organismo sin animo de lucro llamado World Wide Web Consortium, más conocido como W3C.
(https://exprimicompu.blogspot.com/2018/06/Definicion-HTML.html)

CSS

CSS son las siglas de Cascading Style Sheets - Hojas de Estilo en Cascada - que es un lenguaje que describe la presentación de los documentos estructurados en hojas de estilo para diferentes metodos de interpretación, es decir, describe como se va a mostrar un documento en pantalla, por impresora, por voz (cuando la información es pronunciada a través de un dispositivo de lectura) o en dispositivos tactiles basados en Braille. (https://www.masadelante.com/faqs/css#:~:text=CSS%20son%20las%20siglas%20de,impresora%2C%20por%20voz%20).

Carateristicas: Podemos aplicar CSS a muchos niveles, desde un sitio web entero hasta una pequeña etiqueta. Estos son los principales bloques de acción.
1. Un web entero, de modo que se puede definir en un único lugar el estilo de toda una web, de una sola vez.
2. Un documento HTML o página en particular. Se puede definir la forma de cada uno de los bloques de contenido de una página, en una declaración que afectara a un solo documento de un sitio web.
3. Una porcion del documento, aplicando estilos visibles en un trozo de la pagina, como podria ser la cabecera.
4. Una etiqueta en concreto, llegando incluso a poder definir varios estilos diferentes para una sola etiqueta. Esto es muy importante ya que ofrece potencia en nuestra programación. Podemos definir, por ejemplo, varios tipos de párrafos: en rojo, en azul, con margenes, sin ellos.
5. Además podemos definir la distancia entre lineas del documento.
6. Se puede aplicar identado (sangrado) a las primeras lineas del párrafo.
7. Podemos colocar elementos en la página con mayor precisión, y sin lugar a errores.
8. Y mucho más, como definir la visibilidad de los elementos, margenes, subrayados, tachados.
(https://desarrolloweb.com/articulos/181.php)

XHTML

es un lenguaje similar a HTML, ps que lo hacen mas robusto y aconsejable para la modelación de paginas web.
1. Los documentos XHTML son conformes a XML. Como tales, son facilmente visualizados, editados y validados con herramientas XML estandar. pero con algunas diferencia.
2. Los documentos XHTML pueden escribirse para que funcionen igual o mejor que lo hacían antes tanto en las aplicaciones de usuario conformes a HTML 4.0 como en los nuevas aplicaciones conformes a XHTML 1.0.
3. Los documentos XHTML pueden usar aplicaciones (por ejemplo scripts y applets) que se basen ya sea en el Modelo del Objeto Documento de HTML o XML[DOM].
4. A medida que la familia XHTML evolucione, los documentos conformes a XHTML 1.0 estaran mas preparados para interactuar dentro de y entre distintos entornos XHTML.
(https://desarrollowebcarloseduardo.wordpress.com/2017/05/10/caracteristicas-de-xhtml/)

XML

(siglas en inglés de eXtensible Markup Language) es un lenguaje de marcado sencillo similar al HTML. Su objetivo es facilitar la representacion, almacenamiento y trasmision de informacion varia por parte de aplicaciones informaticas, computadoras y medios de comunicacion digital en general. XML no es un lenguaje en particular sino que sirve de marco para definir lenguajes para diferentes necesidades. Algunos ejemplos son XHTML, XSLT, SOAP.
caracteristicas: 1. Permite la creacion de etiquetas propias y permite asignar atributos a las etiquetas.
2. En un documento XML la estructura y el diseno estan completamente separados.
3. XML se almacena en formato texto (no binario) lo cual hace que los documentos sean directamente entendibles. Es decir, los documentos tienen una estructura entendible tanto por los ordenadores como por las personas.
4. Cada documento incluye metadatos sobre si mismo, lo cual facilita la tarea de los motores de búsquedas en la web, ya que devolverán respuestas mas adecuadas y precisas.
5. Permite la exportabilidad a otros formatos de publicacion de datos (HTML, PDF, texto enriquecido RTF, entre otros).
6. XML es un estandar abierto no sujeto a ningun tipo de licencia.
7. XML permite la internacionalización, es decir puede trabajar con cualquier conjunto de caracteres, entre ellos el juego de caracteres UNICODE (utf-8).
8. XML utiliza reglas de generación concretas y, por tanto, los documentos son fácilmente procesables.
9. XML permite compartir información entre sistemas o fuentes de datos heterogeneas, por ejemplo, páginas web, distintas bases de datos, entre otras.
(https://www.ecured.cu/XML#:~:text=XML(siglas%20en%20ingl%C3%A9s%20de,de%20comunicaci%C3%B3n%20digital%20en%20general).

MARK DOWN

es un lenguaje de marcado ligero que trata de conseguir la maxima legibilidad y facilidad de publicación tanto en su forma de entrada como de salida, inspirandose en muchas convenciones existentes para marcar mensajes de correo electronico usando texto plano. Basicamente, Markdown es una herramienta de conversion de texto plano a HTML. Tal cual. Markdown utiliza una sintaxis de formato muy simple para lograr lo mismo que el HTML. La diferencia es que es mas simple que el HTML y no tienes que preocuparte por abrir y cerrar etiquetas. Para dar formato al texto, Markdown utiliza signos de puntuacion y caracteres con los que estamos familiarizados para generar formatos mas basicos.
Por ejemplo, para crear un encabezado se utilizan hashtags. Por lo tanto, # HEADLINE es un encabezado grande; ## HEADLINE sería un poco más pequeño, ### HEADLINE seria aun más pequeño.
(https://www.analiticaweb.es/agilidad-facilidad-markdown/)

XPATH

(XML Path Language) es un lenguaje que permite construir expresiones que recorren y procesan un documento XML. La idea es parecida a las expresiones regulares para seleccionar partes de un texto sin atributos (plain text). XPath permite buscar y seleccionar teniendo en cuenta la estructura jerarquica del XML. Este lenguaje pone a disposición de los usuarios una sintaxis no basada en XML que permite encontrar elementos concretos en un documento XML. Por lo general, Xpath viene integrado en un lenguaje host que permite el procesamiento de los elementos XML. XQuery, por ejemplo, sirve para extraer informacion de elementos XML localizados con Xpath. XSLT, por su parte, se sirve del lenguaje de consulta en la transformación de documentos XML.
-Xpath: navegación en documentos XML
-XQuery: consultas a documentos XML
-XSLT: transformación de datos XML
(https://www.ionos.es/digitalguide/paginas-web/desarrollo-web/tutorial-de-xpath/)

XQUERY

XQUERY es el lenguaje para consultar datos XML.
XQuery para XML es como SQL para bases de datos.
XQuery se basa en expresiones XPath.
XQuery es compatible con las principales bases de datos.
XQuery es una recomendación del W3C.
XQuery es un lenguaje para buscar y extraer elementos y atributos de documentos XML.
Aquí hay un ejemplo de lo que XQuery podría resolver: "Seleccione todos los discos de CD con un precio inferior a $ 10 de la colección de CD almacenada en cd_catalog.xml" XQuery 1.0 y XPath 2.0 comparten el mismo modelo de datos y admiten las mismas funciones y operadores. Si ya ha estudiado XPath, no tendrá problemas para comprender XQuery. XQuery se puede utilizar para: Extraer información para usar en un servicio web
Generar informes resumidos
Transforma datos XML a XHTML
Busque documentos web para obtener información relevante
